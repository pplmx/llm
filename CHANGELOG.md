- **Prefix Tuning multi-backend extension** (T2 PEFT, P2 efficient fine-tuning): closes the gap where Prefix Tuning only supported the MHA backend. Replaces the hard `isinstance(base_attn, MultiHeadAttention)` gate in `PrefixTuningAttention.__init__` with a Protocol-based `isinstance(base_attn, PrefixCapableAttention)` gate ŌĆö any structurally-compliant base now qualifies. An explicit `hasattr` check on `num_kv_heads` and `head_dim` is added on top, because the Protocol's `@runtime_checkable` decorator only inspects method names (so a generic `nn.Linear` would otherwise sneak through the Protocol check and only blow up later when the wrapper reads `num_kv_heads`). `FlashAttention.forward` gains `prefix_kv: tuple[Tensor, Tensor] | None = None` (inserted at the end of the kwarg list, mirroring MHA); prefix is concatenated to K/V after the KV-cache write and before the GQA repeat, the captured `current_kv` (when `use_cache=True`) excludes the prefix ŌĆö matching the MHA contract at `MultiHeadAttention.forward` line 205-206 ŌĆö and the prefix dtype is auto-cast to match the projected K/V dtype to satisfy `flash_attn_func`'s fp16/bf16 requirement. Paged cache + prefix remains impossible on Flash (the existing paged-not-implemented gate fires first). `MultiLatentAttention.forward` gains the same `prefix_kv` parameter; prefix is concatenated to K/V after the cache routing (works with both `KVCache` linear pool and `PagedKVCache` ŌĆö the cache itself stores only dynamic tokens) and before `_latent_attention` runs, so the latent queries attend to the prefix along with the cached context; the `attn_mask` S_k axis is extended by `prefix_len` ones (the prefix is unconditionally visible regardless of the input mask ŌĆö without this widening the latent attention's mask reshape at `_latent_attention` line 169-172 would mismatch the extended K shape). MLA has no GQA so the GQA-repeat step present in MHA/Flash is omitted. - **Pfeiffer Adapter foundation slice** (T2 PEFT #45, P2 efficient fine-tuning): delivers the FFN-only adapter variant from Pfeiffer et al. 2020 ŌĆö the production default in AdapterHub / HuggingFace PEFT, roughly half the parameters of Houlsby 2019 at comparable accuracy on most tasks. The wrapper class is the same `AdapterLinear` used by Houlsby ŌĆö Pfeiffer IS Houlsby-on-MLP-only, so no new tensor code is needed. non-uv users get the same via `[project.optional-dependencies].dvc`. `dvc.yaml` at the repo root declares a minimal `track_dummy_corpus` stage (a no-op template so users can verify the DVC install works end-to-end) with a long comment explaining how to extend it with user-specific dataset acquisition stages (a `vars:` block with `data_fingerprint` / `dedup_algo` so fingerprint changes auto-trigger re-runs; concrete `cmd:` entries). `.dvcignore` excludes the standard non-data paths (`.venv/`, `__pycache__/`, `.pytest_cache/`, `htmlcov/`, `site/`, `checkpoints/`, `results/`, `*.log`, `core.*`) plus DVC's own bookkeeping (`.dvc/cache/`, `.dvc/tmp/`, `.dvc/jobs/`, `.dvc/lock`). 23 tests in `tests/data/test_dvc_integration.py` cover: `compute_fingerprint_hash` returns 64-char hex, identical dicts hash identically, key order is irrelevant (sorted JSON), nested structures round-trip, `pathlib.Path` instances are coerced via `default=str`, empty dict has a deterministic hash; filesystem probes (`is_dvc_initialized`, `dvc_status`) work without DVC installed (the "not installed" path) and correctly distinguish `tracked` / `untracked` / `not_found` / `no_dvc` (the "installed" path); `DVC_AVAILABLE` is a bool; `dvc_add` / `dvc_pull` return `None` / `False` when DVC is missing; `init_dvc` is idempotent + raises on CLI failure with stderr surfaced; `dvc_add` creates the `.dvc` marker + records the fingerprint hash in metadata + is idempotent on already-tracked paths + raises `FileNotFoundError` on missing targets; ### Added

- **`llm-migrate-ckpt` CLI** (P1 ecosystem integration, follow-up to ADR-006): closes the gap that the v2 split-layout checkpoint format (commit `4b9cf68`) opened ŌĆö the `DeprecationWarning` that fires when the `CheckpointManager` loader auto-detects a v0.0.5-era legacy single-file `.pt` checkpoint told users to run `llm-migrate-ckpt <path>`, but the CLI didn't exist. New `src/llm/cli/migrate_ckpt.py` (typer-based, consistent with `llm-train`) exposes a single-command CLI: `llm-migrate-ckpt <path>` (or `<stem>` ŌĆö the function appends `.pt` automatically) converts a legacy `.pt` to the v2 split trio at the same stem (`<stem>.safetensors` + `<stem>.meta.json` + `<stem>.extra_state.pt`), atomically (each sidecar writes to a `.tmp` then renames). Flags: `--in-place` deletes the legacy file after a successful conversion (off by default ŌĆö preserves the legacy blob so the user can verify before deletion); `--verify` re-reads both layouts and round-trip-checks the model_state tensors (with a 1e-5 float-drift tolerance, since safetensors uses a different reduction order than pickle) and metadata fields (`epoch` / `loss` / `best_loss` / `model_config`); `--dry-run` prints the conversion plan without writing anything; `--overwrite` allows replacing an existing split-layout trio (off by default ŌĆö the convert refuses to clobber an already-converted checkpoint). When `--verify` is passed AND a split layout already exists, the CLI skips the conversion step and verifies the existing trio (the "I converted yesterday, did anything drift?" workflow). The conversion logic lives in a public helper `convert_legacy_checkpoint_to_split(path, *, in_place=False, overwrite=False)` in `src/llm/training/core/checkpoint.py`, returning a dict of sidecar paths. A distinct `CheckpointMigrationError` exception type lets callers (CLI, tests) catch migration-specific failures without masking unrelated `RuntimeError`s. The CLI is wired into `[project.scripts]` in `pyproject.toml` so `uv sync` exposes `llm-migrate-ckpt` on `PATH`. 20 tests in `tests/training/test_migrate_ckpt.py` cover: conversion helper ŌĆö three sidecars written, meta.json carries `format_version="2.0"` + epoch + loss + best_loss + model_config, extra_state sidecar carries optimizer + extra_state, model_state tensors round-trip through safetensors, `--in-place` deletes legacy, default keeps legacy, stem path is accepted, missing legacy raises `CheckpointMigrationError("not found")`, refuses when split layout already exists (the existing files are preserved untouched), `--overwrite` replaces stale sidecars, atomic writes leave no `.tmp` residue; CLI surface ŌĆö `--help` exits 0, `--dry-run` writes nothing, happy path writes all three sidecars + keeps legacy, `--in-place` deletes legacy, `--verify` passes for a clean legacy blob, `--verify` exits 2 when meta.json is corrupted post-conversion, missing legacy exits 1, existing split layout without `--overwrite` exits 1, stem path is accepted. Full training suite (280 passed, +20 from this slice) + e2e suite (37 passed) green.
