# Prompt

## Learning Knowledge

### option 1

```txt
请为【填写知识点】生成一份引导式学习文档。
**我的背景**：8年后端经验，精通Python/Go，LLM初学者, 想深入理解LLM。
**文档要求**：
1. **核心概念直觉化**：用生活比喻解释本质，说明解决了什么问题
2. **关键接口设计**：提供清晰的函数/类签名，**只定义不实现**
3. **实现策略**：列出3-5个关键步骤和技术选型考量点
4. **深度思考题**：从理解→设计→扩展三个层次的挑战问题
5. **常见陷阱**：列出典型错误现象，不直接给解法
6. **验证标准**：测试用例设计和性能评估维度
7. **延伸学习**：相关进阶主题和应用场景

**输出风格**：保持引导性，多用启发式问题，避免直接给答案。

```

### option 2

```txt
请为【填写知识点】生成一份深度学习文档。
**我的背景**：8年后端经验，精通Python/Go，**LLM初学者**希望深入理解。

**文档要求**：
1. **概念体系构建**：详细解释核心概念，建立知识框架和概念间的关联
2. **技术原理剖析**：深入的工作机制、算法流程，包含必要的数学原理
3. **发展历程梳理**：技术演进路径，为什么出现，解决了什么问题
4. **关键技术对比**：与相关技术的详细对比分析（优缺点、适用场景）
5. **实现思路解析**：核心算法的具体思路和关键技术难点
6. **深度思考题**：帮助理解本质的启发性问题
7. **常见误区澄清**：初学者容易混淆的概念和典型理解错误
8. **知识扩展链接**：相关的重要概念、前置知识、后续学习方向

**内容要求**：
1. 提供充实的知识密度，确保初学者能建立完整的理解框架。
2. 不要使用中文标点符号, 尽量遵循中文技术文档指北
3. 充分利用 LaTeX 语法
```

### option 3

```txt
请为【填写知识点】生成一份深度学习文档
**我的背景**: 8年后端经验, 精通Python/Go, LLM初学者, 希望深入理解

**输出规则**:
- 文档分为 Part1 和 Part2 两部分生成
- 这次只生成 Part1, 等我回复 "继续" 后, 再生成 Part2
- 每个部分需要有小节和清晰结构, 可用列表或表格组织内容
- 数学部分使用 LaTeX 公式, 并结合伪代码/图解说明工程实现
- 推荐学习资源具体到论文, 教程, 开源实现
- 遵循中文技术文档指北, 不使用中文标点

**文档结构**:
- Part1:
  1. 概念体系构建: 用框架化方式解释核心概念, 说明各部分之间的关联

  4. 关键技术对比: 与相关方法的优缺点对比, 适用场景, 表格化展示

- Part2:
  5. 实现思路解析: 核心算法的实现思路, 关键难点, 工程注意事项
  6. 深度思考题: 提出能帮助理解本质的启发性问题
  7. 常见误区澄清: 初学者容易混淆的概念和典型理解错误
  8. 知识扩展链接: 相关前置知识, 经典论文, 代码库, 后续学习方向

```
